# KadVault iOS App via Capacitor (Live Server)

**Date:** 2026-04-16
**Status:** Approved

## Goal

Wrap the KadVault Next.js PWA in a native iOS shell using Capacitor so it can be distributed via TestFlight. The app loads the deployed Vercel URL in a WKWebView. No changes to existing Next.js code.

## Architecture

```
Next.js app (Vercel) ← WKWebView (Capacitor iOS shell)
                     ← Safari (buyer storefront, unchanged)
```

The iOS app is vendor-only. The buyer storefront remains a plain website accessed via QR code — no app download friction.

## Approach

**Capacitor with Live Server URL.** The iOS app points to the deployed Next.js URL. All app logic stays server-side. Updates deploy via Vercel without App Store resubmission.

### Why This Approach

- Zero code changes to Next.js — SSR, API routes, everything works as-is
- App requires internet anyway (Supabase)
- Hot-deploy updates via Vercel
- Capacitor plugin ecosystem available for future native features

### Trade-offs Accepted

- Requires internet connection (acceptable — Supabase dependency already exists)
- Apple may scrutinize "thin wrapper" apps — mitigated by camera usage and PWA-like native feel
- No offline support for app shell (can add service worker caching later)

## What Changes

| File | Change |
|------|--------|
| `capacitor.config.ts` | New — server URL, app ID, app name |
| `ios/` | Generated by Capacitor — Xcode project |
| `ios/App/App/Info.plist` | Camera permission description |
| `.gitignore` | iOS build artifacts |
| `package.json` | Capacitor dependencies + npm scripts |

**No changes to:** pages, components, API routes, next.config, or any existing code.

## Implementation Steps

1. Install `@capacitor/core`, `@capacitor/cli`, `@capacitor/ios`
2. Create `capacitor.config.ts` with live server URL
3. Run `npx cap add ios` to generate iOS project
4. Add `NSCameraUsageDescription` to Info.plist
5. Generate placeholder app icon (user replaces with real icon)
6. Update `.gitignore`
7. Add npm scripts: `cap:sync`, `cap:open`

## Manual Steps (User)

1. Open Xcode via `npm run cap:open`
2. Select Apple Developer team for signing
3. Product > Archive > Distribute to App Store Connect
4. Create app in App Store Connect, enable TestFlight
5. Invite testers

## Not Included (Future)

- Push notifications (requires APNs + server setup)
- Native camera plugin (browser API sufficient for now)
- Splash screen customization
- Offline support
