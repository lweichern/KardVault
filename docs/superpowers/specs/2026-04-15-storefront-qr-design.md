# Public Storefront + QR Code Generation — Design Spec

## Overview

Two features:

1. **Public storefront page** at `/v/[slug]` — SSR page buyers access via QR code or link. Light theme. Shows vendor info and inventory with search/filter. Displays card name, set, condition, and market price only — never the vendor's asking price. WhatsApp button for inquiries.

2. **QR code on vendor storefront tab** — replace the placeholder at `src/app/(vendor)/storefront/page.tsx` with a live QR code pointing to the vendor's public storefront, plus a shareable link.

## New Files

| File | Purpose |
|------|---------|
| `src/app/v/[slug]/page.tsx` | SSR server component — fetches vendor + inventory, sets OpenGraph meta, renders client component |
| `src/app/v/[slug]/storefront-client.tsx` | Client component — interactive search, set filter chips, WhatsApp buttons, card list |
| `src/app/(vendor)/storefront/page.tsx` | Rewrite — QR code display, copy link, share, stats |

## Modified Files

| File | Change |
|------|--------|
| `next.config.ts` | Add `images.remotePatterns` for `images.pokemontcg.io` |

## Dependencies to Install

- `qrcode` — QR code generation (renders to data URL)
- `@types/qrcode` — TypeScript types

## Public Storefront Page

### Data Flow (SSR)

The page is a **server component**. On each request:

1. Extract `slug` from URL params
2. Fetch vendor from `vendors` table where `slug = params.slug`
3. If vendor not found → Next.js `notFound()`
4. Fetch inventory with card join: `inventory.select("*, card:cards(*)").eq("vendor_id", vendor.id)`
5. Pass vendor + inventory items to the client component

The server component also sets dynamic metadata (OpenGraph) for social sharing.

### Metadata

```ts
export async function generateMetadata({ params }) {
  // Fetch vendor by slug
  // Return: title: "{vendor.display_name}'s Store | KardVault"
  // description: "Browse {count} Pokémon TCG cards"
  // openGraph image: KardVault default OG image
}
```

### Client Component Layout (Light Theme — matches CLAUDE.md Screen 8)

The client component uses the `storefront-*` color tokens already defined in `globals.css`.

**Top-to-bottom layout:**

1. **Vendor header**
   - Avatar circle: vendor initials on violet gradient background (`bg-primary-400`)
   - Display name (16px bold, `storefront-text`)
   - Bio text (11px, `storefront-text2`), if present

2. **Stats row**
   - Cards count (from inventory length)
   - Sets count (unique set names from inventory)

3. **WhatsApp button**
   - Full-width, green (`#25D366`) background, WhatsApp icon + "Message on WhatsApp"
   - Opens: `https://wa.me/{vendor.whatsapp_number}?text=Hi, I found your store on KardVault!`
   - The whatsapp_number is fetched server-side and passed as a prop (the `vendors` table RLS allows public SELECT)

4. **Search bar**
   - Light input bg (`storefront-input`), magnifying glass icon
   - Client-side filtering of the inventory list (no API calls — all data is pre-fetched via SSR)

5. **Filter chips**
   - "All" chip active by default (`storefront-chip-active` bg with white text)
   - One chip per unique set name from the inventory
   - Horizontal scroll, outline style when inactive

6. **Card list**
   - Each card row: card image (50×70px), card name (13px medium), set info, condition badge (violet pill)
   - Right side: "Available" label (green text), "Market" label (tiny uppercase) + market price in RM (14px bold)
   - **No vendor asking price shown anywhere**
   - Tapping a card row opens WhatsApp: `https://wa.me/{whatsapp}?text=I'm interested in {card.name} from your KardVault store`

7. **Footer**
   - "Powered by KardVault" watermark (`storefront-powered-by` color)
   - Free tier only (for now, always show it since Pro tier isn't implemented)

### Condition Badge Colors

Same as existing inventory page:
- NM → green
- LP → blue
- MP → yellow
- HP → orange/red
- DMG → red

### WhatsApp Number Formatting

Malaysian WhatsApp numbers stored as `+60123456789`. The `wa.me` link uses the number without the `+`: `https://wa.me/60123456789`.

### Empty Inventory State

If vendor has 0 inventory items, show a centered message: "No cards listed yet" with a card icon.

### Vendor Not Found

Return Next.js `notFound()` → shows default 404 page.

## QR Code on Vendor Storefront Tab

### Rewrite `src/app/(vendor)/storefront/page.tsx`

This is a client component. It uses `useAuth`, `useVendor`, and `useInventory` hooks.

**Layout:**

1. **Header**: "KardVault" logo + "Your Storefront" subtitle
2. **QR Code card** (centered):
   - QR code rendered as an `<img>` from a data URL generated by the `qrcode` library
   - QR encodes: `{window.location.origin}/v/{vendor.slug}`
   - Size: 200×200px
   - Below QR: short URL text `kardvault.com/v/{slug}` (or dynamic domain)
3. **Action buttons** (two columns):
   - "Copy Link" — copies the storefront URL to clipboard, shows brief "Copied!" feedback
   - "Open Storefront" — opens the `/v/{slug}` page in a new tab
4. **Stats card**:
   - Cards listed count
   - Sets count
   - "Your storefront is live" status indicator (green dot) when inventory > 0

### QR Code Generation

```ts
import QRCode from "qrcode";

const url = `${window.location.origin}/v/${vendor.slug}`;
const dataUrl = await QRCode.toDataURL(url, {
  width: 200,
  margin: 2,
  color: { dark: "#1A1825", light: "#FFFFFF" },
});
```

Generated once on mount via `useEffect`, stored in state as a data URL string.

### Empty State

If vendor has no inventory, show the QR code anyway but with a note: "Add cards to your inventory so buyers have something to browse."

## Next.js Image Config

Add remote pattern for card images from pokemontcg.io:

```ts
const nextConfig: NextConfig = {
  turbopack: {},
  images: {
    remotePatterns: [
      { protocol: "https", hostname: "images.pokemontcg.io" },
    ],
  },
};
```

## Out of Scope

- Pro tier branding (no watermark removal, no vendor logo/banner on storefront)
- Vendor ratings/reviews
- Cross-vendor search
- Event mode
- Social sharing meta image generation (use static KardVault OG image)
